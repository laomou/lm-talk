# 存储规格 v1

Web 客户端使用 IndexedDB 分表保存本地状态。敏感字段使用应用层 AES-GCM 加密。

## 加密字段

应加密保存：

- 消息文本和 envelope；
- 联系人显示名和名片；
- 群名和群策略；
- outbox 载荷；
- Ratchet 会话状态；
- 好友请求和群邀请原文；
- 群 Sender Key 状态和分发文本；
- Mailbox 失败队列载荷和原因；
- 同步服务地址和令牌配置。

## 明文字段

为保证索引和 UI 可用，以下字段可以明文保存：

- User ID；
- group id；
- message id；
- 时间戳；
- 状态枚举；
- 计数和布尔标志。

## 迁移要求

- 旧 localStorage 迁移到 IndexedDB 时，必须先成功写入新分表，再删除旧状态。
- 单条联系人、消息、群、outbox 或 Ratchet 记录损坏时，不得阻止整个身份登录。
- 客户端应跳过损坏记录，并提示用户可从备份或同步服务恢复。

## 重加密

用户修改提示词或重加密身份备份后，客户端必须重新派生本地存储密钥，并重写已保存的敏感字段。

## 删除身份

删除本地身份时，必须清理该 User ID 前缀下的 IndexedDB 分表记录，避免仅删除登录入口而保留本机加密聊天数据。
