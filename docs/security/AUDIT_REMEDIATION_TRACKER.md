# 审计修复跟踪

本文用于记录内部或外部安全审查发现。当前功能目标不要求第三方审计；该表作为可选跟踪模板保留。

## 状态值

- `Open`：已记录，尚未修复。
- `In progress`：正在修复。
- `Fixed`：已修复并验证。
- `Accepted risk`：接受残余风险。
- `Duplicate`：与其他发现重复。
- `Won't fix`：不适用或不修复，需说明原因。

## 严重性

- `Critical`：可导致 E2EE 明文、身份私钥或远程代码执行。
- `High`：严重绕过信任、设备撤销、strict E2EE、认证或持久化保护。
- `Medium`：有意义的安全降级，但需要前提条件或影响有限。
- `Low`：硬化、诊断、日志或深度防御问题。
- `Info`：文档、流程或非安全建议。

## 每个发现建议记录

- 受影响组件：core、wasm、web、node、deploy、ci、docs。
- 威胁模型和利用叙述。
- 重现步骤或 PoC。
- 期望安全属性。
- 实际行为。
- 修复计划。
- 修复提交或接受理由。
- 验证命令或测试。
- 审核人。

## 使用方式

该跟踪表用于帮助后续安全审查和维护，不再作为当前功能目标阻塞项。若未来要做公开生产发行，可将本表与发布证据一起归档。
