# 安全策略

LM Talk 是一个端到端加密、本地优先的即时通讯项目。欢迎安全报告，但在发布检查清单中的阻塞项关闭之前，项目不应被视为生产就绪。

## 支持的版本

此仓库尚未达到 1.0 版本。除非明确宣布发布分支，否则安全修复范围仅限当前的 `main`/`master` 分支。

## 如何报告漏洞

请私下报告安全问题。请勿公开提交包含利用细节、密钥、令牌、备份、消息明文或其他敏感材料的问题。

请尽可能包括以下内容：

- 受影响的组件：`lm_core`、`lm_wasm`、Web UI/PWA、`lm_node`、部署脚本或文档。
- 影响范围和所需攻击者能力。
- 针对最小本地环境的重现步骤。
- 该问题是否暴露明文、身份种子材料、备份文本、承载令牌、本地 IndexedDB 数据、节点 `state_db`、DHT 记录或邮箱元数据。
- 相关提交哈希和平台/浏览器/操作系统。

如果需要包含日志，请先删除或编辑以下内容：

- 身份备份和数据备份；
- 密码短语、种子材料、私钥和设备私钥材料；
- 承载令牌和 `http://host|token` 同步 URL；
- 消息明文、附件内容和完整协议负载，除非确实必要。

## 预期响应

对于私下报告，维护者应确认收到、分类严重性并协调修复，然后再公开披露。如果维护者不可用，请避免在用户尚未有合理时间更新之前发布可行的漏洞利用细节。

## 已知的非生产阻塞项

以下是已知的阻塞项，除非您有具体的利用或绕过方案，否则无需作为新漏洞报告：

- 尚未完成外部安全审计。
- 尚未完成具有已保存语料库和崩溃分类的长期模糊测试活动。
- 原生节点 `state_db` 是明文 SQLite，而不是 SQLCipher/加密。节点在此构建中显示 `state_db_encrypted=false`；`state_db_require_encryption=true` 会故意失败关闭。
- DHT/Kademlia 仍然是一个强化的骨架，而不是具有抗 Sybil、节点评分、alpha 并发和公开部署模型的完全生产级公共 DHT。
- 中继/TURN 替代方案和公共中继滥用控制尚未达到生产完整性。
- Web/PWA 后台事件不会在 Service Worker 中解密或同步消息；用户必须打开应用才能使用本地密钥。
